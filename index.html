What was built

A full-feature delivery: 8 distinctly-themed 3D platforming stages (Meadow → Desert → Frost Peak → Jungle → Coral Isles → Mine → Sky → Volcano Castle), one end-of-stage boss each, a full transformation set, a title/level-select screen, DOM HUD + tutorial, per-stage BGM, a complete feedback layer for damage / pickups / combos / clears, plus a batch of non-infringing Sketchfab asset swaps. Everything is originalized (renamed, generic art); it does not reproduce Nintendo IP.

Pitfalls hit

Editor collaboration / deploy format fork

Trigger: after the user edits in the editor, cloud head leads local; local is CLI format (inline scene-identity), cloud clone is editor format (_scenes.json + res_*Model).

Symptom: deploy returns 500 / "changed upstream".

Fix: deploy from the local CLI tree; decide "did cloud lead because the user changed content" via diff scene.instances.ts == 0 and an empty overrides shell → pure normalization, safe to push.

Custom GLB injection engine

Trigger: referencing a Sketchfab GLB directly. Symptom: a TANGENT attribute crashes dev WebGPU; importing a multi-element pack whole clumps all 6 models together.

Fix: ① python-strip the TANGENT attribute per primitive and rewrite the JSON chunk; ② extract a single mesh by named node; ③ content-address by sha256 → aws s3 cp s3://wanaka-dev/assets/ → reference via asset:.

Extracted single mesh still ~20MB

Cause: only the node graph was changed, the BIN buffer wasn't trimmed, so the whole pack's shared textures ride along. State: kept (low-frequency item, runs); can later trim the buffer to a few hundred KB.

Far plane clipping the backdrop

After adding distant peaks / clouds / landmarks they got clipped → set c.camera.farPlane = 16000 each frame.

set() used before declaration / boss maxHp re-declared

Caught by: the Wanaka compile gate errored directly. Fix: move the call after the const; hoist the boss variables and drop the duplicate const in the fire block.

"Deaths out of nowhere"

Grazing the edge of lava / quicksand killed you → tightened the hazard test to "horizontally very close + height near the pool surface".

Real-window automation can't read state engine

Cause: the Preview game runs in a cross-origin iframe; the nested WebGPU canvas doesn't composite in the automation browser → black screen (fine on the user's own machine). Fallback: compile gate + build-mode screenshots confirm rendering; playability/feel left for a live pass.

Gaps vs the original

  • Fine-grained level design: the 8 stages are structurally distinct, but per-stage difficulty curve, hidden areas and reachability tuning still need repeated live play — currently "structure in place, feel to be tuned".
  • Camera / final feel: the follow-cam was nudged conservatively; the exact distance/pitch sweetness needs a call on real hardware.
  • Player character asset: still the Wanaka rigged character (swapping to a static Sketchfab model would lose the run/jump animations); no original character yet.
  • Some items / the bell: no clean non-infringing bell was found, so the library asset stays; mega / propeller / tanuki items unchanged.
  • Swap orientation / scale: coin / star / crystal sizes were estimated from thumbnails and may need a live tweak.
  • Looks like the original: deliberately not a 1:1 copy of Nintendo's characters / levels / assets — this is a high-quality same-genre original, not a clone.

How to play the demo

W A S Dmove K / ␣jump (chain = triple jump) Jsprint / attack when transformed Lcrouch · in air = ground pound Rrestart

Grab coins (100 = an extra life) and Power Stars, stomp / hit enemies (chained stomps score a combo), hit crates for coins and chests for power-ups, dodge hazards — defeat the end-of-stage boss (read its wind-up warning and dodge sideways), then reach the goal flag to clear, advancing stage by stage to the Volcano Castle showdown with the Flame Dragon. Lose a life → checkpoint; out of lives → title / level-select.
