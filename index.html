踩的坑

编辑器协作 / 部署格式分叉

触发：用户在编辑器手改后云端 head 领先本地；本地是 CLI 格式（内联 scene-identity），云端 clone 是编辑器格式（_scenes.json + res_*Model）。

验证：deploy 报 500 / “changed upstream”。

修复：从本地 CLI 树部署；判定「云端领先=是否用户改内容」用 diff scene.instances.ts == 0 且 overrides 空壳 → 纯归一化，直接推安全。

自定义 GLB 注入 引擎层

触发：Sketchfab GLB 直接引用。现象：TANGENT 属性令 dev WebGPU 崩；多元素包整包导入 6 模型糊一起。

修复：① python 读 GLB 删每 primitive 的 TANGENT 重写 JSON chunk；② 按命名 node 抽单 mesh；③ 内容寻址 sha256 → aws s3 cp s3://wanaka-dev/assets/ → asset: 引用。

抽单 mesh 后文件仍 ~20MB

根因：只改 node 图没裁 BIN 缓冲，整包共享贴图仍在。现状：保留（道具低频、能跑），可后续裁 buffer 到几百 KB。

相机远裁面裁掉远景

加背景（远山/云/地标）后远处被裁 → 每帧 c.camera.farPlane = 16000。

set() 声明前使用 / Boss maxHp 重复声明

发现：Wanaka compile 编译门直接报错拦下。修复：挪声明顺序、提函数顶变量、删块内重复 const。

「莫名其妙死亡」

路过岩浆/流沙边缘就掉命 → 危险区判定收紧为「水平很近 + 高度贴近池面」才掉命。

真窗口自动化读不到状态 引擎层

根因：Preview 游戏跑在跨域 iframe，自动化浏览器里嵌套 WebGPU 画布不合成 → 黑屏（用户自己机器正常）。兜底：编译门 + build 截图确认渲染，可玩性/手感最终需真机验。

与原作差距

  • 精细关卡设计：8 关结构各异，但布局难度曲线、隐藏区、可达性微调需真机反复试玩 —— 目前「结构到位、手感待调」。
  • 相机 / 手感最终体感：follow 相机保守调过，具体 distance/pitch 的爽度要真机拍板。
  • 玩家角色资产：仍用 Wanaka 带动画角色（换 Sketchfab 静态模型会丢跑跳动画），未做原创角色。
  • 部分道具 / 铃铛：铃铛没找到干净非侵权模型，保留库资产；巨化 / 竹蜻蜓 / 狸猫未换。
  • 换装朝向 / 尺寸：金币 / 星 / 水晶按缩略图估的，真机可能要微调。
  • 像不像原作：刻意不 1:1 抄任天堂角色 / 关卡 / 资产 —— 是「同类型优质原创」，非复制品。
